## 1. 落地内容

- **配置面**:`QueueConfig` 加 `QuotaLimit`(0=不启用)/`QuotaPeriod`(≥1s)/`QuotaKey`(空=队列名,同键共享预算);`validate()` fail-fast(负值/缺 period/同键参数冲突),`New()` 能力断言(后端不实现 `QuotaProvider` 而配了配额 → 直接报错,**没有静默降级**)。
- **能力接口**:`QuotaProvider` / `QuotaGate`(Reserve 三态:成功 / (nil,nil) 耗尽 / err 介质故障)/ `QuotaReservation{Window}`,与 `LimiterProvider` 平行;Broker 接口本体零改动。
- **五后端原子预留**("检查+扣减"无窗口,服务端时间):memory 单锁+注入钟;sqlite 单语句 `INSERT..ON CONFLICT..WHERE used<? RETURNING`(原型同款+COALESCE 测试缝);PG 同构单语句(`EXTRACT(EPOCH FROM now())`);MySQL 两步(取时算窗 + 单条 ODKU 按 affected 判定——取时分步不破坏硬性,原子性在扣减上);redis 单段 Lua(`TIME` 是"禁 TIME"约定的唯一豁免,INCR+EXPIRE 同脚本无锁死窗口)。窗口轮换时机会主义清理旧行(redis 靠 EXPIRE 自清)。
- **调度接线**:认领链四环——占槽 → 令牌 → `QueueLen>0` 启发式 + 预留 → 限时 Dequeue;耗尽/介质故障先释放槽再退避(`min(period/8,1s)`,注入 clock);扑空尽力退还,退不掉当 leaked(保守);fail-closed 零放行。`QueueStats` 加 `QuotaExhausted`/`QuotaStalled` 两个观测位。
- **测试**:新 `brokertest.RunQuota` 合规套件(Q1 硬上限/Q2 窗口轮换/Q3 旧窗退还落空/Q4 键隔离与共享/Q5 并发竞态),五后端接入(介质时间:fakeclock / SetTestQuotaNow 钩子 / miniredis SetTime,全程不真 sleep);L3(耗尽调度/双 Gate 共享 sqlite 文件/stub 闸 fail-closed);L4 e2e 三维组合(`{Workers:2,RPS:8,Quota:10/2s}`,窗口 1 恰好 10 个、并发 ≤2)。

## 3. 基准数据(FR-013 / SC-007,Apple M 系 10 核,-benchtime 1s)

| 基准 | 数值 |
|---|---|
| sqlite Reserve(单发热点行) | 27.4 µs/op |
| sqlite Reserve(32 并发热点行) | 30.8 µs/op(单写者串行,几乎无退化) |
| sqlite Enqueue→Dequeue→Ack 无配额 | 202 µs/op |
| sqlite 同上 + 每次认领一次预留 | 228 µs/op(**损耗 +12.4%**) |
| MySQL 真机 Reserve(单发热点行) | 547 µs/op(含网络往返,与认领事务同数量级) |
| MySQL 真机 Reserve(32 并发热点行) | 153 µs/op(吞吐 ~6.5k ops/s,热点行不构成瓶颈) |
| miniredis 档 | 数字不代表真 Redis(gopher-lua 解释开销主导),真 redis 的预留是一段小 Lua,与既有 enqueue 脚本同量级;真机档留待 `TASKGATE_REDIS_ADDR` 冒烟 |

**判定**:损耗远低于 30% 触发线(sqlite 实测 +12.4%),**不触发备选路线**(Broker 原子 admission/claim)重评。
